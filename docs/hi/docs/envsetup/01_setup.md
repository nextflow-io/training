# GitHub Codespaces

GitHub Codespaces एक वेब-आधारित प्लेटफ़ॉर्म है जो हमें क्लाउड में वर्चुअल मशीनों द्वारा समर्थित, प्रशिक्षण के लिए एक पूर्व-कॉन्फ़िगर किया गया वातावरण प्रदान करने की अनुमति देता है।
यह प्लेटफ़ॉर्म GitHub (जो Microsoft के स्वामित्व में है) द्वारा संचालित है, और GitHub अकाउंट वाले किसी भी व्यक्ति के लिए मुफ़्त में (उपयोग कोटा के साथ) सुलभ है।

!!! warning "चेतावनी"

    संगठनों से जुड़े अकाउंट कुछ अतिरिक्त प्रतिबंधों के अधीन हो सकते हैं।
    यदि यह तुम्हारा मामला है, तो तुम्हें एक स्वतंत्र व्यक्तिगत अकाउंट का उपयोग करने की आवश्यकता हो सकती है, या इसके बजाय लोकल इंस्टॉलेशन का उपयोग करना होगा।

## GitHub अकाउंट बनाना

तुम [GitHub होम पेज](https://github.com/) से एक मुफ़्त GitHub अकाउंट बना सकते हो।

## अपना GitHub Codespace लॉन्च करना

एक बार जब तुम GitHub में लॉग इन हो जाओ, तो Nextflow प्रशिक्षण वातावरण खोलने के लिए अपने ब्राउज़र में यह लिंक खोलो: <https://codespaces.new/nextflow-io/training?quickstart=1&ref=master>

वैकल्पिक रूप से, तुम नीचे दिखाए गए बटन पर क्लिक कर सकते हो, जो प्रत्येक प्रशिक्षण कोर्स में दोहराया जाता है (आमतौर पर Orientation पेज पर)।

[![Open in GitHub Codespaces](https://github.com/codespaces/badge.svg)](https://codespaces.new/nextflow-io/training?quickstart=1&ref=master)

तुम्हें एक पेज दिखाई देना चाहिए जहाँ तुम एक नया GitHub Codespace बना सकते हो:

![Create a GitHub Codespace](img/codespaces_create.png)

### कॉन्फ़िगरेशन

सामान्य उपयोग के लिए, तुम्हें कुछ भी कॉन्फ़िगर करने की आवश्यकता नहीं होनी चाहिए।
जब तक कोर्स में जो तुम शुरू कर रहे हो अन्यथा निर्दिष्ट न हो, तुम बस जारी रखने के लिए मुख्य बटन पर क्लिक कर सकते हो।

हालाँकि, "Change options" बटन पर क्लिक करके वातावरण को कस्टमाइज़ करना संभव है।

??? info "कॉन्फ़िगरेशन विकल्प"

    यदि तुम "Change options" बटन पर क्लिक करते हो, तो तुम्हें निम्नलिखित को कस्टमाइज़ करने का विकल्प दिया जाएगा:

    #### Branch

    यह तुम्हें प्रशिक्षण सामग्री का एक अलग संस्करण चुनने की अनुमति देता है।
    `master` ब्रांच में आमतौर पर बग फिक्स और सामग्री होती है जो हाल ही में विकसित और अनुमोदित की गई है लेकिन अभी तक वेबसाइट पर रिलीज़ नहीं की गई है।
    अन्य ब्रांचों में प्रगति पर काम होता है जो पूरी तरह से कार्यात्मक नहीं हो सकता है।

    #### Machine type

    यह तुम्हें उस वर्चुअल मशीन को कस्टमाइज़ करने की अनुमति देता है जिसका उपयोग तुम प्रशिक्षण के माध्यम से काम करने के लिए करोगे।

    अधिक कोर वाली मशीन का उपयोग करने से तुम Nextflow की workflow निष्पादन को समानांतर करने की क्षमता का अधिक लाभ उठा सकते हो।
    हालाँकि, यह तुम्हारे मुफ़्त कोटा आवंटन को तेज़ी से खपत करेगा, इसलिए हम इस सेटिंग को बदलने की सिफारिश नहीं करते हैं जब तक कि तुम जो कोर्स लेने की योजना बना रहे हो उसके निर्देशों में इसकी सलाह न दी गई हो।

    कोटा के बारे में अधिक विवरण के लिए नीचे 'GitHub Codespaces quotas' देखो।

### स्टार्टअप समय

पहली बार एक नया GitHub Codespaces वातावरण खोलने में कई मिनट लग सकते हैं, क्योंकि सिस्टम को तुम्हारी वर्चुअल मशीन सेट अप करनी होती है, इसलिए चिंता मत करो अगर प्रतीक्षा समय है।
हालाँकि, इसमें पाँच मिनट से अधिक नहीं लगना चाहिए।

## प्रशिक्षण इंटरफ़ेस में नेविगेट करना

एक बार जब तुम्हारा GitHub Codespaces लोड हो जाए, तो तुम्हें निम्नलिखित के समान कुछ दिखाई देना चाहिए (जो तुम्हारे अकाउंट प्राथमिकताओं के आधार पर लाइट मोड में खुल सकता है):

![GitHub Codespaces welcome](img/codespaces_welcome.png)

यह VSCode IDE का इंटरफ़ेस है, एक लोकप्रिय कोड डेवलपमेंट एप्लिकेशन जिसे हम Nextflow डेवलपमेंट के लिए उपयोग करने की सिफारिश करते हैं।

- **मुख्य एडिटर** वह जगह है जहाँ Nextflow कोड और अन्य टेक्स्ट फ़ाइलें खुलेंगी। यह वह जगह है जहाँ तुम कोड एडिट करोगे। जब तुम codespace खोलते हो, तो यह तुम्हें `README.md` फ़ाइल का एक प्रीव्यू दिखाएगा।
- मुख्य एडिटर के नीचे **टर्मिनल** तुम्हें कमांड चलाने की अनुमति देता है। यह वह जगह है जहाँ तुम कोर्स निर्देशों में दी गई सभी कमांड लाइनें चलाओगे।
- **साइडबार** तुम्हें अपने वातावरण को कस्टमाइज़ करने और बुनियादी कार्य करने की अनुमति देता है (कॉपी, पेस्ट, फ़ाइलें खोलना, खोज, git, आदि)। डिफ़ॉल्ट रूप से यह फ़ाइल एक्सप्लोरर के लिए खुला होता है, जो तुम्हें रिपॉज़िटरी की सामग्री ब्राउज़ करने की अनुमति देता है। एक्सप्लोरर में किसी फ़ाइल पर क्लिक करने से वह मुख्य एडिटर विंडो में खुल जाएगी।

तुम विंडो पेन के सापेक्ष अनुपात को अपनी पसंद के अनुसार समायोजित कर सकते हो।

## GitHub Codespaces का उपयोग करने के बारे में अन्य नोट्स

### एक सत्र फिर से शुरू करना

एक बार जब तुमने एक वातावरण बना लिया है, तो तुम आसानी से इसे फिर से शुरू या पुनरारंभ कर सकते हो और जहाँ से तुमने छोड़ा था वहाँ से जारी रख सकते हो।
तुम्हारा वातावरण 30 मिनट की निष्क्रियता के बाद टाइम आउट हो जाएगा और 2 सप्ताह तक तुम्हारे परिवर्तनों को सहेज लेगा।

तुम <https://github.com/codespaces/> से एक वातावरण को फिर से खोल सकते हो।
पिछले वातावरण सूचीबद्ध होंगे।
इसे फिर से शुरू करने के लिए एक सत्र पर क्लिक करो।

![List GitHub Codespace sessions](img/codespaces_list.png)

यदि तुमने अपने पिछले GitHub Codespaces वातावरण के लिए URL सहेजा है, तो तुम बस इसे अपने ब्राउज़र में खोल सकते हो।
वैकल्पिक रूप से, उसी बटन पर क्लिक करो जिसका उपयोग तुमने पहली बार इसे बनाने के लिए किया था:

[![Open in GitHub Codespaces](https://github.com/codespaces/badge.svg)](https://codespaces.new/nextflow-io/training?quickstart=1&ref=master)

तुम्हें पिछला सत्र दिखाई देना चाहिए, डिफ़ॉल्ट विकल्प इसे फिर से शुरू करना है:

![Resume a GitHub Codespace](img/codespaces_resume.png)

### अपनी लोकल मशीन पर फ़ाइलें सहेजना

एक्सप्लोरर पैनल से किसी भी फ़ाइल को सहेजने के लिए, फ़ाइल पर राइट-क्लिक करो और `Download` चुनो।

### GitHub Codespaces कोटा का प्रबंधन

GitHub Codespaces तुम्हें प्रति माह 15 GB-month स्टोरेज, और प्रति माह 120 कोर-घंटे देता है।
यह स्टैंडर्ड वर्कस्पेस (2 कोर, 8 GB RAM, और 32 GB स्टोरेज) का उपयोग करते हुए डिफ़ॉल्ट वातावरण रनटाइम के लगभग 60 घंटे के बराबर है।

तुम उन्हें अधिक संसाधनों के साथ बना सकते हो (ऊपर स्पष्टीकरण देखो), लेकिन यह तुम्हारे मुफ़्त उपयोग को तेज़ी से खपत करेगा और तुम्हारे पास इस स्पेस तक पहुँच के कम घंटे होंगे।
उदाहरण के लिए, यदि तुम 2-कोर डिफ़ॉल्ट के बजाय 4-कोर मशीन चुनते हो, तो तुम्हारा कोटा आधे समय में समाप्त हो जाएगा।

वैकल्पिक रूप से, तुम अधिक संसाधनों तक पहुँच खरीद सकते हो।
