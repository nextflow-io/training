## लेसन प्लान

डेवलपमेंट प्रोसेस को अधिक शैक्षिक बनाने के लिए, हमने इसे {N} स्टेप्स में विभाजित किया है:

1. **एक सिंगल-स्टेज वर्कफ़्लो लिखो जो {TOOL_A_ACTION} चलाता है।**
   यह एक मॉड्यूल बनाना, उसे इम्पोर्ट करना, और वर्कफ़्लो में कॉल करना कवर करता है।
2. **{TOOL_B_ACTION} चलाने के लिए दूसरा प्रोसेस जोड़ो।**
   यह प्रोसेस आउटपुट्स को इनपुट्स से चेन करना और एक्सेसरी फ़ाइलों को हैंडल करना परिचित कराता है।
3. **वर्कफ़्लो को सैंपल्स के बैच पर चलाने के लिए अनुकूलित करो।**
   यह पैरेलल एक्ज़ीक्यूशन को कवर करता है और संबंधित फ़ाइलों को एक साथ रखने के लिए टपल्स को परिचित कराता है।
4. **वर्कफ़्लो को एक सैंपलशीट स्वीकार करने योग्य बनाओ जिसमें इनपुट फ़ाइलों का बैच हो।**
   यह बल्क में इनपुट प्रदान करने के लिए एक सामान्य पैटर्न को प्रदर्शित करता है।

प्रत्येक स्टेप वर्कफ़्लो डेवलपमेंट के एक विशिष्ट पहलू पर केंद्रित है।

हम इस जानकारी को तीन स्टेज्स में Nextflow में रैप करने जा रहे हैं:

1. इनपुट सेट अप करो
2. प्रोसेस लिखो और इसे वर्कफ़्लो में कॉल करो
3. आउटपुट हैंडलिंग कॉन्फ़िगर करो

### 1.1. इनपुट सेट अप करो

हमें एक इनपुट पैरामीटर डिक्लेयर करना होगा, एक सुविधाजनक डिफ़ॉल्ट वैल्यू प्रदान करने के लिए एक टेस्ट प्रोफ़ाइल बनानी होगी, और एक इनपुट चैनल बनाना होगा।

#### 1.1.1. एक इनपुट पैरामीटर डिक्लेरेशन जोड़ो

मुख्य वर्कफ़्लो फ़ाइल `{DOMAIN_DIR}.nf` में, `Pipeline parameters` सेक्शन के अंतर्गत, `{PRIMARY_PARAM_NAME}` नामक एक CLI पैरामीटर डिक्लेयर करो।

=== "बाद में"

    ```groovy title="{DOMAIN_DIR}.nf" linenums="5" hl_lines="4-7"
    /*
     * Pipeline parameters
     */
    params {
        // Primary input
        {PRIMARY_PARAM_NAME}: Path
    }
    ```

=== "पहले"

    ```groovy title="{DOMAIN_DIR}.nf" linenums="5"
    /*
     * Pipeline parameters
     */

    // Primary input
    ```

यह CLI पैरामीटर सेट अप करता है, लेकिन हम डेवलपमेंट के दौरान हर बार वर्कफ़्लो चलाते समय फ़ाइल पाथ टाइप नहीं करना चाहते।
डिफ़ॉल्ट वैल्यू प्रदान करने के लिए कई विकल्प हैं; यहाँ हम एक टेस्ट प्रोफ़ाइल का उपयोग करते हैं।

#### 1.1.2. `nextflow.config` में डिफ़ॉल्ट वैल्यू के साथ एक टेस्ट प्रोफ़ाइल बनाओ

एक टेस्ट प्रोफ़ाइल कमांड लाइन पर इनपुट्स निर्दिष्ट किए बिना वर्कफ़्लो को आज़माने के लिए सुविधाजनक डिफ़ॉल्ट वैल्यूज़ प्रदान करती है।
यह Nextflow इकोसिस्टम में एक सामान्य कन्वेंशन है (अधिक विवरण के लिए [Hello Config](../../hello_nextflow/06_hello_config.md) देखो)।

`nextflow.config` में एक `profiles` ब्लॉक जोड़ो जिसमें एक `test` प्रोफ़ाइल हो जो `{PRIMARY_PARAM_NAME}` पैरामीटर को टेस्ट {PRIMARY_INPUT_TYPE}s में से एक पर सेट करे।

=== "बाद में"

    ```groovy title="nextflow.config" linenums="1" hl_lines="3-7"
    docker.enabled = true

    profiles {
        test {
            params.{PRIMARY_PARAM_NAME} = "${projectDir}/data/{TEST_INPUT_PATH}"
        }
    }
    ```

=== "पहले"

    ```groovy title="nextflow.config" linenums="1"
    docker.enabled = true
    ```

यहाँ, हम `${projectDir}` का उपयोग कर रहे हैं, एक बिल्ट-इन Nextflow वेरिएबल जो उस डायरेक्टरी को पॉइंट करता है जहाँ वर्कफ़्लो स्क्रिप्ट स्थित है।
यह एब्सोल्यूट पाथ्स को हार्डकोड किए बिना डेटा फ़ाइलों और अन्य रिसोर्सेज़ को रेफरेंस करना आसान बनाता है।
