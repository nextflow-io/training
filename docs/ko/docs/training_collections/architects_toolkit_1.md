# The Architect's Toolkit I

우리의 교육 컬렉션은 고급 교육 자료([Side Quests](../../side_quests)라고 함)를 통한 선별된 학습 경로를 제공합니다. 이 컬렉션은 강력하고 확장 가능한 워크플로우를 구축하는 데 자주 함께 사용되는 네 가지 필수 주제를 다룹니다.

## 학습 목표

이 컬렉션을 마치면 다음과 같은 경험을 얻게 됩니다:

- **복잡한 모듈식 워크플로우 아키텍처** - 여러 워크플로우를 응집력 있는 파이프라인으로 결합하기
- **포괄적인 테스트 전략** - 워크플로우의 신뢰성과 유지보수성 보장하기
- **메타데이터 관리** - 워크플로우 전반에 걸쳐 샘플별 메타데이터를 효과적으로 처리하기
- **고급 데이터 처리** - 효율적인 데이터 분할 및 그룹화 패턴 구현하기

이러한 기술을 통해 실제 애플리케이션을 위한 강력하고 확장 가능하며 유지보수 가능한 Nextflow 워크플로우를 구축할 수 있습니다.

## 대상 및 사전 요구 사항

이 컬렉션은 기본 Nextflow 교육을 완료하고 고급 워크플로우 패턴, 테스트 전략, 데이터 및 메타데이터 처리 기법을 더 깊이 학습하고자 하는 사용자를 위해 설계되었습니다.

**사전 요구 사항**

- [Hello Nextflow](../../hello_nextflow/) 교육 완료 또는 동등한 경험
- Nextflow 구문 및 개념에 대한 기본적인 이해
- 기본 워크플로우 개발 패턴에 대한 이해
- 명령줄 도구 사용 경험

## 컬렉션 내용

이 컬렉션은 상호 보완적인 워크플로우 엔지니어링 주제를 다루는 네 개의 Side Quest로 구성됩니다:

1. **[Workflows of Workflows](../../side_quests/workflows_of_workflows)** - 복잡한 워크플로우 아키텍처 및 구성
2. **[Testing with nf-test](../../side_quests/nf-test)** - Nextflow 워크플로우를 위한 테스트 전략
3. **[Metadata](../../side_quests/metadata)** - Nextflow 채널의 항목에 대한 메타데이터 처리
4. **[Splitting and Grouping](../../side_quests/splitting_and_grouping)** - 고급 데이터 처리 패턴

각 Side Quest는 독립적인 개념을 다루는 자체 완결형이지만, 주제의 논리적인 진행을 위해 위에 나열된 순서대로 완료하는 것을 권장합니다.

## 이 컬렉션 사용 방법

먼저, 아래의 "Open in GitHub Codespaces" 버튼을 Ctrl+클릭(또는 Cmd+클릭)하여 별도의 탭에서 교육 환경을 실행한 다음, 로딩되는 동안 계속 읽어주세요.

[![Open in GitHub Codespaces](https://github.com/codespaces/badge.svg)](https://codespaces.new/nextflow-io/training?quickstart=1&ref=master)

환경이 실행되면 다음과 같이 컬렉션을 진행하세요:

1. 이 탭에서: 위에 나열된 첫 번째 Side Quest로 이동합니다. 단계별 개발 연습이 설명되어 있습니다.
2. Codespaces 탭에서: Side Quest의 연습 문제를 진행합니다.
3. Side Quest를 완료하면 이 페이지로 돌아와 위 목록의 다음 항목으로 이동합니다.
4. 컬렉션을 모두 완료하면 아래 버튼을 클릭하여 매우 짧은 설문조사를 작성해 주세요. 여러분의 피드백은 모두를 위한 교육 자료를 지속적으로 개선하는 데 도움이 됩니다.
