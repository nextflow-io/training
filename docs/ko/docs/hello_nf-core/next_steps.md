## 여러분의 여정

데모 파이프라인을 가져와서 실행하는 방법을 배우는 것으로 시작하여, 간단한 Nextflow 워크플로우를 nf-core 파이프라인으로 변환하는 작업을 수행했습니다.
템플릿을 사용하여 파이프라인 스캐폴드를 생성하는 방법을 배우고, 기존 파이프라인을 해당 스캐폴드에 접목했습니다.
그런 다음 로컬 모듈 중 하나를 nf-core 모듈로 교체하고, 다른 로컬 모듈을 nf-core 표준에 맞게 변환하며, 입력 검증을 추가하여 파이프라인을 단계적으로 개선했습니다.

### 구축한 결과물

최종 `core-hello` 파이프라인은 다음을 갖추고 있습니다:

- **표준화된 구조**: nf-core 템플릿을 사용하여 워크플로우, 서브워크플로우, 모듈, 구성을 위한 체계적인 디렉토리 구조
- **커뮤니티 모듈**: 사용자 정의 모듈과 함께 nf-core 저장소의 모듈(`cat/cat`) 통합
- **포괄적인 검증**: 파이프라인 실행 전 매개변수와 입력 데이터를 모두 확인
- **전문적인 구성**: 다양한 실행 환경을 위한 프로파일
- **완전한 문서화**: nf-core 규칙을 따르는 메타데이터

### 습득한 핵심 기술

이 실습 과정을 통해 다음을 배웠습니다:

1. 기존 파이프라인을 탐색하여 nf-core 파이프라인 구조를 **탐색하고 이해**하기
2. 워크플로우를 구성 가능하도록 **재구성**하고 nf-core 템플릿에 맞추기
3. 커뮤니티 저장소에서 사전 구축된 모듈을 **찾아서 통합**하기
4. 이름 지정, 구조, 메타데이터에 대한 nf-core 표준을 따르는 **사용자 정의 모듈 생성**하기
5. nf-schema를 사용하여 명확한 피드백과 함께 오류를 조기에 포착하는 **검증 구현**하기

이제 커뮤니티 모범 사례를 따르는 프로덕션 수준의 nf-core 파이프라인을 구축할 수 있는 기초 지식을 갖추게 되었습니다.

## 기술 향상을 위한 다음 단계

다음 단계로 권장하는 상위 3가지 제안입니다:

- [Nextflow for Science](../nf4_science/index.md)로 과학적 분석 사용 사례에 Nextflow 적용하기
- [Side Quests](../side_quests/index.md)로 더 고급 Nextflow 기능 탐색하기

마지막으로, Nextflow 창시자들이 개발한 클라우드 기반 플랫폼인 [**Seqera Platform**](https://seqera.io/)을 살펴보시기를 권장합니다. 이 플랫폼을 사용하면 워크플로우를 더욱 쉽게 시작하고 관리할 수 있으며, 데이터를 관리하고 모든 환경에서 대화형으로 분석을 실행할 수 있습니다.
