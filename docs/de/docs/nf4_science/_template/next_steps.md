## Deine Lernreise

Du hast damit begonnen, {METHOD}-Tools manuell im Terminal auszuführen, um die Methodik zu verstehen.
Dann hast du eine Nextflow-Pipeline für eine einzelne Probe erstellt, um den Prozess zu automatisieren, sie auf die parallele Verarbeitung mehrerer Proben skaliert und {AGGREGATION_SUMMARY} für mehrere Proben hinzugefügt.

### Was du erstellt hast

- Eine {METHOD}-Pipeline, die {PRIMARY_INPUT_TYPE}s als Eingabe nimmt und {FINAL_OUTPUT_TYPE}s als Ausgabe erzeugt.
- {N} Prozesse ({PROCESS_LIST}), die in separaten Moduldateien gespeichert sind.
- Die Pipeline skaliert automatisch auf eine beliebige Anzahl von Eingabeproben mithilfe des Dataflow-Paradigmas von Nextflow.
- Die Ergebnisse werden in einem Verzeichnis namens `results/` veröffentlicht.

### Erworbene Fähigkeiten

Durch diesen praxisorientierten Kurs hast du gelernt, wie du:

- Einen linearen Workflow schreibst, um {METHOD} auf eine einzelne Probe anzuwenden
- Zusatzdateien wie {ACCESSORY_FILES} angemessen handhabst
- Das Dataflow-Paradigma von Nextflow nutzt, um die Verarbeitung pro Probe zu parallelisieren
- Aggregation mehrerer Proben mithilfe relevanter Channel-Operatoren implementierst
